# OctoAcme Project Management Docs

Welcome! This README provides a concise overview of how OctoAcme manages projects and links to the detailed process documents stored in this docs/ folder. Use this as a starting point to onboard team members, find process checklists, and locate deeper guidance for each phase of the project lifecycle.

## Summary of Project Management Processes
OctoAcme follows an iterative, customer-focused approach with lightweight, documented processes to reduce cognitive load and improve repeatability. Projects progress through five main stages: Initiation, Planning, Execution, Release, and Retrospective. At each stage teams produce specific artifacts (one-pager, backlog, risk register, release notes, retrospective action items) and follow defined checklists.

Key characteristics:
- Customer-first: prioritize delivering measurable customer value.
- Iterative delivery: break work into small, testable increments and deliver frequently.
- Clear ownership: projects have named PM and Product Lead roles.
- Data-informed decisions: define success metrics and measure impact.
- Psychological safety & transparent communication: encourage candid feedback and blameless retrospectives.

## High-level Lifecycle
1. Initiation — capture problem, stakeholders, goals, and a one-pager to decide go/no-go.
2. Planning — define scope, backlog, estimates, DoD, dependencies, and release milestones.
3. Execution — build, test, review, and track work on the project board; escalate blockers as needed.
4. Release — follow the release checklist, run smoke tests, and verify post-deploy health.
5. Retrospective — capture lessons, create action items, and iterate on process improvements.

## Key Roles (at a glance)
- Project Manager (PM): delivery coordination, risks, schedule, and communication.
- Product Manager (PdM): defines outcomes, prioritizes, and measures success.
- Developers: implement and test features.
- QA/Testing: verify acceptance criteria and quality.
- Stakeholders: provide inputs and approvals.
