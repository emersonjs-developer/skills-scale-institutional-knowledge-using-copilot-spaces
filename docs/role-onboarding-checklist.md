# Role Onboarding Checklist

A reusable checklist template for onboarding any role on an OctoAcme project. Copy this template for each new role-holder and fill in the project-specific details.

---

## Template

**Role:** `<role name>`
**Project:** `<project name>`
**Onboarding date:** `<date>`
**Buddy/point of contact:** `<name>`

### Week 1 — Introductions & Access

- [ ] Meet with PM and PdM for project overview
- [ ] Meet with direct collaborators (list them here)
- [ ] Gain access to required tools (list tools here — e.g., Jira, GitHub, Slack, Figma)
- [ ] Added to relevant communication channels (list channels)
- [ ] Assigned read access to key docs (one-pager, roadmap, backlog)

### Week 1–2 — Key Docs Review

- [ ] Read the Project Charter / One-pager
- [ ] Review the current Roadmap and Release Plan
- [ ] Review the Sprint/Iteration Backlog
- [ ] Read the [Roles & Personas guide](octoacme-roles-and-personas.md)
- [ ] Read the [Role Interactions & Accountability Guide](octoacme-role-interactions.md)
- [ ] Review the Risk Register

### Week 2 — Communication Cadence & Decision Rights

- [ ] Understand the weekly meeting schedule (standups, planning, reviews, retros)
- [ ] Confirm which meetings you own, facilitate, or attend
- [ ] Clarify your decision rights (what you can decide alone vs. what needs escalation)
- [ ] Confirm your reporting expectations (to whom, how often, in what format)

### 30-Day Goal

> `<Define one specific, measurable outcome to achieve in the first 30 days>`

- [ ] Goal documented and agreed with PM/PdM
- [ ] First deliverable or milestone identified

### 60-Day Goal

> `<Define a broader contribution or process improvement target by day 60>`

- [ ] Goal documented
- [ ] Key dependencies or blockers identified and tracked

### 90-Day Goal

> `<Define a strategic contribution or maturity milestone for day 90>`

- [ ] Goal documented
- [ ] Progress check-in scheduled with PM

### Stakeholder List

- [ ] Reviewed the stakeholder list with PM
- [ ] Identified which stakeholders you interact with directly
- [ ] Confirmed preferred communication styles for key stakeholders

### Reporting Expectations

- [ ] Frequency and format of status updates confirmed with PM
- [ ] Know where to log risks, decisions, and blockers
- [ ] Understand escalation path (see [Role Interactions & Accountability Guide](octoacme-role-interactions.md))

## Example: Scrum Master Onboarding

**Role:** Scrum Master
**Project:** OctoAcme Customer Portal v2
**Onboarding date:** 2026-04-01
**Buddy/point of contact:** Alex (Project Manager)

### Week 1 — Introductions & Access

- [x] Met with Alex (PM) and Jordan (PdM) for project overview on Day 1
- [x] Met with dev team leads and QA lead
- [x] Gained access to Jira, GitHub, and Slack (#portal-delivery, #portal-standup)
- [x] Added to sprint board and retrospective tool (RetroBoard)
- [x] Read current sprint backlog and sprint goal

### Week 1–2 — Key Docs Review

- [x] Read the Project Charter for Customer Portal v2
- [x] Reviewed the Roadmap (Q2 milestones noted)
- [x] Read octoacme-roles-and-personas.md
- [x] Read octoacme-role-interactions.md
- [ ] Risk Register review scheduled for Week 2 sync with PM

### Week 2 — Communication Cadence & Decision Rights

- [x] Confirmed: facilitates daily standup (15 min), sprint planning (2 hrs), retro (1 hr)
- [x] Attends but does not facilitate: stakeholder demos (PM leads)
- [x] Decision rights: can re-order sprint tasks and declare blockers; scope changes escalated to PM
- [x] Reporting: sends sprint burndown summary to Alex (PM) at end of each sprint

### 30-Day Goal

> Run three sprints end-to-end with consistent ceremonies and measurable reduction in standup blockers.

- [x] Goal agreed with Alex (PM)
- [x] First sprint planning session facilitated

### 60-Day Goal

> Establish a team working agreement and retrospective action-item tracking process.

- [ ] Working agreement drafted
- [ ] Retro action-item board created in Jira

### 90-Day Goal

> Demonstrate velocity stability (±10% variance) over four consecutive sprints and present findings at a team retrospective.

- [ ] Velocity tracking set up in Jira
- [ ] Check-in with PM scheduled for Week 12

### Stakeholder List

- [x] Reviewed stakeholder list with Alex (PM)
- [x] Identified: dev team, QA, PdM, and Alex (PM) as primary collaborators
- [x] Stakeholder demo audience noted; communication goes through PM

### Reporting Expectations

- [x] Sprint burndown + brief summary → Alex (PM) at end of each sprint
- [x] Blocker log maintained in Jira; critical blockers escalated to PM within 24 hrs
- [x] Retrospective action items logged and owned by team; tracked in Jira
